# BabyGuard

Cross-platform baby monitor built with Flutter + Firebase.
One phone becomes the **Baby Unit** (listens for sound in the background); the other becomes the **Parent Unit** (receives loud, full-screen alerts).

---

## Quick commands

Once your toolchain is set up (see [§ 1a](#1a-build-from-zero-on-macos-apple-silicon)) and Firebase config is in place ([§ 3](#wire-firebase-credentials)), drive everything through the `Makefile`:

```bash
make help        # list every target
make ship        # build debug APK and install on every attached device
make run         # flutter run on the first attached device (hot reload)
make logs        # tail flutter + errors from the phone
make logs-fn     # tail Cloud Function logs
make deploy-fn   # deploy Firestore rules + Cloud Functions
make test        # widget smoke tests
make analyze     # static analysis
```

Common day-to-day loop: edit code → `make ship` → test on phones. For a backend change: edit `functions/index.js` → `make deploy-fn` → `make logs-fn`.

Ready to ship to the Play Store? Run `make aab` to produce a signed App Bundle, then walk through:

- [docs/firebase_upgrade.md](docs/firebase_upgrade.md) — re-register the renamed Android app in Firebase, harden Firestore rules, enable App Check, restrict the API key.
- [docs/googleplay_setup.md](docs/googleplay_setup.md) — Play Console account, App Signing, sensitive-permission declarations, listing assets, internal → production promotion ladder.

## Project structure

```
lib/
├── core/        constants, theme, routing, utilities (pair codes, permissions)
├── services/    firebase, pairing, FCM, notifications, noise meter, background task
├── providers/   Riverpod state: auth, pairing, monitoring, settings
├── features/    UI: onboarding, pairing, baby_unit, parent_unit, settings
├── widgets/     shared widgets (BigButton, StatusPill, AppScaffold)
├── app.dart     MaterialApp + Material 3 theme + routes
└── main.dart    Firebase + notifications init, FCM background handler

functions/       Node.js Cloud Function that fans out FCM alerts
firebase.json    Firestore + Functions config
firestore.rules  Starter security rules
android/         Manifest with all required permissions
```

> **State management** uses Riverpod 2 — chosen because it works *outside* the widget tree (needed for the foreground-service isolate and FCM background handler) and is compile-safe.

## 1. Prerequisites

- Flutter ≥ 3.22 (Dart ≥ 3.4)
- Node.js 20 + `firebase-tools` (`npm i -g firebase-tools`)
- Two Android devices (real hardware — emulators have no microphone)
- A Firebase project on the **Blaze** plan (free Spark plan does not allow outbound FCM via Cloud Functions)

## 1a. Build from zero on macOS (Apple Silicon)

This is the exact sequence used to produce a working `app-debug.apk` from a clean macOS install. If you already have Flutter + Android SDK + Java 17 wired up, skip to section 2.

### One-time toolchain install

```bash
# Flutter SDK
brew install --cask flutter

# Java 17 — needed by Gradle. Use the keg-only formula (no sudo required).
brew install openjdk@17
export JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home
export PATH="$JAVA_HOME/bin:$PATH"

# Android SDK command-line tools
brew install --cask android-commandlinetools
export ANDROID_HOME=/opt/homebrew/share/android-commandlinetools
export PATH="$ANDROID_HOME/cmdline-tools/latest/bin:$ANDROID_HOME/platform-tools:$PATH"

# Accept Android licenses + install SDK packages Flutter 3.41 expects
yes | sdkmanager --licenses
sdkmanager "platform-tools" \
  "platforms;android-34" "platforms;android-35" "platforms;android-36" \
  "build-tools;34.0.0" "build-tools;35.0.0" "build-tools;36.0.0"

# Point Flutter at the SDK
flutter config --android-sdk "$ANDROID_HOME"
flutter doctor
```

Persist the `JAVA_HOME` / `ANDROID_HOME` / `PATH` exports in your `~/.zshrc` so future shells inherit them.

### Project-specific scaffolding (only on a fresh clone)

The repository ships with `lib/`, the custom `AndroidManifest.xml`, and the Cloud Functions sources, but the `android/` Gradle scaffolding, `ios/` Xcode project, and `test/widget_test.dart` are generated by `flutter create`:

```bash
git clone https://github.com/Hundd/BabyGuard.git
cd BabyGuard

# Generates android/, ios/, test/ — does NOT overwrite existing pubspec.yaml,
# AndroidManifest.xml, or lib/ files.
flutter create --project-name baby_guard --org com.example --platforms=android,ios .
```

### Wire Firebase credentials

Two files are required and both are gitignored:

```bash
# 1. Copy the schema and fill in real values
cp firebase-config.example.json firebase-config.json
# Edit firebase-config.json — values come from your Firebase console
# (Project settings → Your apps → Android app → SDK setup and configuration).

# 2. Drop google-services.json into android/app/
#    Firebase console → Project settings → Your apps → Android app → google-services.json
```

`firebase-config.json` is fed into Dart at compile time via `--dart-define-from-file`; `lib/firebase_options.dart` reads each value with `String.fromEnvironment`. `google-services.json` is required by the `com.google.gms.google-services` Gradle plugin and must live at `android/app/google-services.json`.

### Build the APK

```bash
flutter pub get
flutter analyze        # should print "No issues found" (info-level lints ok)
flutter test           # smoke-builds the widget tree
flutter build apk --debug --dart-define-from-file=firebase-config.json
```

Output: `build/app/outputs/flutter-apk/app-debug.apk` (~170 MB, debug symbols included).
First build is slow (~5 min on M-series — Gradle warms its cache, downloads NDK, accepts more licenses); subsequent builds are 30–60 s.

### Install on a phone

```bash
adb devices                                                # confirm phone is paired
adb install build/app/outputs/flutter-apk/app-debug.apk
```

> The app will launch but crash at the splash with a Firebase error until you complete sections 2–4 (Firebase project + `flutterfire configure`).

### Common gotchas

| Symptom                                              | Cause / Fix                                                                                       |
|------------------------------------------------------|---------------------------------------------------------------------------------------------------|
| `Unable to locate Android SDK`                       | Run `flutter config --android-sdk "$ANDROID_HOME"`.                                              |
| `Unable to locate a Java Runtime`                    | `JAVA_HOME` not exported in this shell — re-source `~/.zshrc`.                                   |
| `Flutter requires Android SDK 36`                    | `sdkmanager "platforms;android-36" "build-tools;36.0.0"`.                                        |
| `sudo: a password is required` during `brew install` | You picked a cask that runs an installer pkg — use `brew install openjdk@17` instead of the Temurin cask. |
| Gradle hangs on first build                          | It's downloading the Android NDK (~1 GB). Wait it out.                                            |

## 2. Firebase Console setup

1. **Create a project** named `babyguard-app` (or similar) at <https://console.firebase.google.com>.
2. **Authentication → Sign-in method → Anonymous → Enable**.
3. **Firestore Database → Create database** in production mode. Pick a region close to you.
4. **Cloud Messaging** is enabled by default.
5. Upgrade to **Blaze** plan so Cloud Functions can send FCM messages.

---

## 3. Wire Firebase into the Flutter app

```bash
# install the FlutterFire CLI
dart pub global activate flutterfire_cli

# from the project root
flutterfire configure --project=<your-project-id>
```

This will:
- Register Android & iOS apps in the Firebase project.
- Drop `google-services.json` into `android/app/`.
- Drop `GoogleService-Info.plist` into `ios/Runner/`.
- **Overwrite** `lib/firebase_options.dart` with real keys (the file currently committed is a placeholder).

---

## 4. Android build configuration

Open `android/app/build.gradle` (generated by `flutter create`) and merge the snippet at `android/app/build.gradle.snippet`:

- Add `com.google.gms.google-services` plugin.
- `minSdkVersion 23` (required by `noise_meter` and `mobile_scanner`).
- `compileSdk 34`, Java/Kotlin 17.
- Enable `coreLibraryDesugaring` (required by `awesome_notifications`).
- Add `coreLibraryDesugaring 'com.android.tools:desugar_jdk_libs:2.0.4'` to `dependencies`.

Open `android/build.gradle` and ensure `classpath 'com.google.gms:google-services:4.4.2'` is in the buildscript dependencies (the CLI may add this automatically).

`AndroidManifest.xml` (already provided at `android/app/src/main/AndroidManifest.xml`) declares:

- `RECORD_AUDIO`, `POST_NOTIFICATIONS`, `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_MICROPHONE`, `USE_FULL_SCREEN_INTENT`, `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`, `WAKE_LOCK`, `VIBRATE`, `CAMERA`.
- The `flutter_foreground_task` service with `foregroundServiceType="microphone"` (required on Android 14+).
- `MainActivity` flagged `showWhenLocked="true"` and `turnScreenOn="true"` so the alert screen wakes the device.

## 5. Deploy the Cloud Function

```bash
cd functions
npm install
cd ..
firebase deploy --only firestore:rules,functions
```

`functions/index.js` defines `onAlertEvent` — a Firestore-trigger on `pairs/{pairId}/events/{eventId}`. When the Baby Unit writes an event, the function reads the pair's `parentToken` and sends a high-priority FCM with `channelId: alert_channel`. The Parent Unit's notification handler (`lib/services/fcm_service.dart`) renders a full-screen alert via `awesome_notifications`.

## 7. Test with two phones

1. **Phone A → Baby Unit**
   - Pick **Baby Unit** on the role selection screen.
   - Grant **Microphone** and **Notifications** when prompted.
   - Approve the **battery optimization** exemption dialog.
   - You'll see a 6-character code + QR code.
2. **Phone B → Parent Unit**
   - Pick **Parent Unit**.
   - Either scan the QR or tap *Enter code* and type the 6 characters.
   - Both screens flip to **Connected / Paired**.
3. **Phone A** → tap **Start Monitoring**.
   - The persistent notification "BabyGuard is monitoring" appears.
   - The on-screen dB meter responds when you talk or clap.
4. Clap loudly near phone A. Within ~1 second phone B should:
   - Wake the screen.
   - Show a red full-screen alert.
   - Play the system **alarm** ringtone and vibrate continuously.
   - **STOP ALERT** silences and dismisses.
5. **Background test**: lock phone A. Repeat the clap — alert still fires.
6. **Sensitivity test**: open *Settings* on phone A → raise threshold to 90 dB → normal speech no longer triggers.
7. **Sanity test** without the baby unit: open *Settings* on phone B → tap **Send Test Alert** to see the alarm pipeline in isolation.

## 8. iOS notes (future)

iOS is supported as a code path but requires additional setup we did not automate in this starter:

- `ios/Runner/Info.plist` keys:
  - `NSMicrophoneUsageDescription` — "BabyGuard listens for sounds in the nursery."
  - `NSCameraUsageDescription` — "Used to scan the pairing QR code."
  - `UIBackgroundModes`: `audio`, `fetch`, `remote-notification`.
- Background microphone on iOS requires the `audio` background mode and a strong App Review justification. Consider migrating to `CallKit` / VoIP background mode for a production release.
- **Critical alerts** (loud alarm sound even on silent mode) require requesting an Apple entitlement. Without it, alerts use normal high-priority push.

## 9. Troubleshooting

| Symptom                                            | Fix                                                                                                                                |
|----------------------------------------------------|------------------------------------------------------------------------------------------------------------------------------------|
| dB always reads 0                                  | Microphone permission denied. Toggle in OS Settings → Apps → BabyGuard.                                                            |
| Foreground notification but no alerts on Parent    | Confirm the Cloud Function deployed (`firebase functions:log`) and that `pairs/{pairId}.parentToken` exists in Firestore.          |
| Parent phone doesn't wake from sleep               | Check `USE_FULL_SCREEN_INTENT` is granted (Android 14: Settings → Apps → BabyGuard → "Full screen notifications") and battery-opt is disabled. |
| Service killed when screen turns off               | Re-prompt for battery optimization exemption (Settings → Unpair → re-pair to retrigger the prompt) or whitelist BabyGuard manually. |
| `noise_meter` crash on emulator                    | Emulators don't expose a microphone reliably. Use real hardware.                                                                   |
